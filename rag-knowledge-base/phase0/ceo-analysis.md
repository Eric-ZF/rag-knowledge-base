# CEO 产品方向分析 — 知研（Zhiyan）

> 为高校讲师和研究者打造的文献研究工作台，让 AI 成为"读过你所有文献的研究伙伴"

**当前阶段**：Phase 0.6（验证期） → Phase 1（原型演示）

---

## 一、用户验证（真实访谈数据）

**访谈对象**：高校讲师

### Q&A 关键数据

| 问题 | 回答 | 洞察 |
|-----|------|------|
| 人工精读 vs GPT 处理 | 1小时/篇 vs 10分钟/篇 | GPT 已有 6x 效率提升 |
| GPT 10篇限制的麻烦 | 多次上传连贯性弱、整合能力差 | **明确的痛点窗口** |
| 100篇存在哪？ | app/服务自带存储，不用额外软件 | 产品本身要做存储载体 |
| 文献组织方式 | 本地 PDF，无 Zotero 等工具 | 从零建立秩序，无迁移成本 |
| 跨论文整合指什么？ | 对比方法论/模型/数据/理论基础；生成文献综述 | **核心差异化功能** |
| 产品形态 | 网页（PC 端） | 不做 App |
| 文献语言 | 中英混排 | embedding 要双语 |
| 输出用途 | 自己用 + 发期刊 | 产出质量要达到发表级别 |
| 输出格式 | 混合（报告 + 可追问） | 文档优先 + 对话追问 |

**核心用户画像：李讲师**
- 高校讲师，每周阅读 10-20 篇论文
- 处理中英混合文献（各约占一半）
- 本地 PDF 存储，无引用管理工具
- 不喜欢额外软件，产品本身要成为存储载体
- 文献综述用途：自己复习 + 投期刊

## 三、竞争分析

| 方案 | 局限 |
|-----|------|
| 人工精读 | 1小时/篇，100篇 = 4天 |
| ChatGPT | 10篇上限，多篇割裂，无结构 |
| Zotero/Mendeley | 仅管理，无 AI 分析能力 |
| **知研** | 不限量 + 跨论文分析 + 期刊级产出 |

**核心差异化 vs GPT**：
- ✅ 不限量（不是 10 篇）
- ✅ 跨论文方法论审计（不是单篇 Q&A）
- ✅ 输出可直接用于期刊（不是对话碎片）
- ✅ 中英双语支持

## 四、产品定位

**产品名称（建议）**：知研（Zhiyan）— "知识研究伴侣"

**一句话定位**：
> 为高校讲师和研究者打造的文献研究工作台，让 AI 成为"读过你所有文献的研究伙伴"，不只是搜索工具。

**旧思维 vs 新思维**：
```
旧思维：上传论文 → 问问题 → 找答案
新思维：上传 100 篇 → 让 AI 成为"读过所有论文的研究员" → 问它任何跨论文问题
```

---

## 五、核心功能（MVP 三件套）

### P0 — 批量上传 + 云端存储
- 不限量上传（区分于 GPT 的 10 篇限制）
- app 内直接存储，不再需要 Zotero/Mendeley
- 后台异步处理，大论文集不用等待

### P0 — 跨论文方法论审计
- 问："哪些论文用了实证模型？用了哪些数据？"
- 问："对比这 5 篇论文的方法论差异"
- 输出：带原文引用的结构化答案，可直接用于文献综述

**典型用例**（讲师原话）：
> "对比不同的方法论/模型，不同的数据来源，不同的理论基础等，对多篇指定/同一研究方向论文，生成整合的文献综述"

### P0 — 文献综述生成
- 指定研究方向或选定 N 篇论文
- 输出：Markdown 格式，可复制到 Word/LaTeX
- 追问："这个结论的原文出处是什么？"

## 七、竞争优势护城河

**短期（难复制）**：
- 论文数据飞轮：越多用户 → 越多论文数据 → 越准的分析
- 学术场景优化的 Prompt 模板（方法论审计、综述生成）

**长期（护城河）**：
- 研究者网络效应：课题组共享同一知识库
- 领域模型积累：成为某个领域（碳中和/环保）最准的文献分析工具
